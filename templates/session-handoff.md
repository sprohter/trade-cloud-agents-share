# 会话交接模板

当当前会话过长且任务尚未完成时，使用本模板做精简交接。内容应简洁、基于证据，并便于粘贴到新会话继续处理。

## 触发规则

满足任一情况时，应生成交接摘要并建议新会话继续：

- 任务已跨过一个明确阶段，但还需要继续实现、验证或复核。
- 会话已经多次“继续 / 接着做”，上下文明显膨胀，开始反复读取 routing、contract、完整报告或历史协作记录。
- 工具输出、日志、SQL、报告或截图证据较多，继续留在同一会话会让下一轮请求携带大量历史材料。
- 单个会话 token 或 JSONL 明显偏大；可用 `.\.agents\scripts\trade.ps1 audit context` 做本地只读审计。
- 需要切换宿主、子 agent、人工接手，或当前会话即将进入另一个独立工作阶段。

交接摘要只保留结论、证据路径、已完成和待完成事项，不复制完整报告原文。

## 当前目标

- 当前正在完成什么任务？

## 已确认事实

- 只列出已通过代码、日志、SQL 结果、截图、配置或工具输出验证的事实。
- 明确区分已确认事实和推断。

## 关键证据

- 文件：`absolute\path\to\file`
- 代码位置：`absolute\path\to\file:line`
- SQL / 表证据：表名、关键条件和结果摘要
- 日志 / 截图 / 配置：路径或来源，以及关键发现

## 已完成工作

- 已经检查、查询或排除了什么？

## 待完成工作

- 还有什么没有完成？
- 当前是否存在阻塞点？

## 下一步动作

- 写出继续推进任务时最应该做的一步。

## 可直接粘贴的提示词

```text
请基于下面的交接内容继续这个未完成任务。

任务目标：
- ...

已确认事实：
- ...

关键证据：
- ...

已完成工作：
- ...

待完成工作：
- ...

下一步动作：
- ...

要求：
- 默认使用中文。
- 先查后说。
- 区分已确认事实和推断。
- 优先给文件路径和简洁证据引用，不重复粘贴大段原文。
```
