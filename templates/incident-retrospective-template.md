# 事故正式复盘模板

## 1. 目的

用于输出结构统一的正式复盘稿，覆盖严重等级、时间线、直接原因、根因和防再发策略。

默认原则：

- 分类不是重点，治理才是重点
- 复盘不是为了证明谁错了，而是为了让同类问题更难再次发生
- 正式复盘必须尽量回答：产品质量哪里不够可靠、团队能力哪里要补、未知缺口哪里要补成机制

详细口径参考：

- [retrospective-principles.md](../knowledge/testing/incident-retrospectives/retrospective-principles.md)

## 2. 适用场景

适用：

- 事故已基本恢复
- 需要提交复盘报告
- 需要准备复盘分享

## 4. 使用边界

正式复盘稿应尽量区分：

- 现象
- 直接原因
- 根因
- 防再发策略

并且要避免：

- 只做分类，不落治理动作
- 只写责任，不写能力和机制补位
- 只解释已知问题，不补未知缺口
不能把四者混在一起写成一段。
