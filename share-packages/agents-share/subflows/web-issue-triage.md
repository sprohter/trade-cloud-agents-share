# Web 问题分诊与起手排查

> 本文件统一 Web 功能异常的起手排查路径。

---

## 1. 一句话原则

先确认异常属于：

- 展示问题
- 接口问题
- 数据问题
- 链路问题

再决定往哪一层继续查。

---

## 2. 起手先回答 4 个问题

1. 页面到底哪里异常
2. 是展示层异常，还是操作后结果异常
3. 页面有没有发请求，请求有没有返回
4. 返回结果和日志 / 数据是否一致

---

## 3. 四类问题的快速判断

| 类型 | 常见特征 | 优先看什么 |
|------|---------|-----------|
| 展示问题 | 接口返回对，页面展示错 | 接口返回、前端渲染、字段映射 |
| 接口问题 | 请求没发对、接口报错、返回结构异常 | 请求参数、响应、服务日志 |
| 数据问题 | 页面和接口都正常，但底层数据本身不对 | MySQL / Mongo、任务执行、操作日志 |
| 链路问题 | 页面提示成功，但后续没有真正生效 | 日志、消息、任务、最终结果数据 |

---

## 4. 什么时候优先抓接口

以下场景优先抓接口：

- 页面显示和预期明显不一致
- 页面操作后没反应
- 页面提示成功但结果不对
- 需要判断是前端问题还是后端问题

重点关注：

- 请求参数
- 返回 code / msg
- 返回数据结构
- 页面展示是否和返回一致

---

## 5. 什么时候优先查日志或数据

以下场景优先往后看：

- 接口已经返回成功，但结果不落地
- 页面只是把错误数据展示出来
- 涉及任务、MQ、平台回写、异步补偿

---

## 6. 默认输出最小集

一次分诊至少建议输出：

- 问题现象
- 当前分类
- 已核对的证据
- 下一步建议
